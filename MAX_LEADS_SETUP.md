# Заявки с сайта → мессенджер MAX: полная инструкция

Документ для быстрого переноса схемы на другие проекты и для ИИ-агента: порядок настройки, контракт API, деплой секретов, **спам-фильтр** (копировать логику, а не изобретать заново).

---

## 1. Как это устроено

| Слой | Назначение |
|------|------------|
| **Фронт** (React и т.п.) | `POST` JSON на `/api/submit-lead.php`, honeypot-поле `company`, поля `name`, `phone`, `comment`, `page`. |
| **Хостинг** (PHP) | `public/api/submit-lead.php` — валидация, антиспам, вызов MAX API. |
| **Секреты** | `api/lead-secrets.php` рядом со скриптом — **не в git**. Токен и `user_id` / `chat_id`. |
| **MAX Platform API** | `POST https://platform-api.max.ru/messages?user_id=…` с заголовком `Authorization: <токен>`. |

Заявки **не блокируются** из‑за спама: в MAX уходит то же сообщение, но с префиксом `⚠️ **SPAM** — …`. Honeypot заполненный ботом — **тихий успех** (`ok: true` без отправки в MAX).

---

## 2. Последовательность в кабинете MAX (business.max.ru)

1. Создать/выбрать **чат-бота** для бизнеса.
2. Пройти модерацию, если требуется.
3. **Интеграция** → получить **токен бота** (для API).
4. С аккаунта, **куда должны приходить уведомления**, открыть бота в приложении MAX и нажать **Старт** / написать `/start` (иначе бот может не иметь права писать первым — смотрите актуальные правила в доках MAX).

Документация (актуализировать ссылки при смене портала):

- [dev.max.ru — API](https://dev.max.ru/docs-api)
- [Подготовка бота](https://dev.max.ru/docs/chatbots/bots-coding/prepare)
- [POST /messages](https://dev.max.ru/docs-api/methods/POST/messages)
- [GET /updates](https://dev.max.ru/docs-api/methods/GET/updates) — отладка и получение `user_id`

## 3. Узнать `user_id` получателя (один раз с ПК)

Подставьте токен из **Интеграция** (без префикса `Bearer`, если в доках указан просто токен — как в вашем рабочем `curl`):

```bash
curl -s -H "Authorization: ВАШ_ТОКЕН_БОТА" \
  "https://platform-api.max.ru/updates?limit=20"
```

В JSON смотрите события `bot_started`, `message_created`: у **пользователя** (не бота) поле вроде `user_id`. Его записывают в `notify_user_id`.

**Альтернатива:** групповой чат с ботом → в ответах может понадобиться `chat_id`; в PHP уже поддерживается `notify_chat_id`, если `notify_user_id = 0` (см. `submit-lead.php`).

|------|------|

## 5. Формат `lead-secrets.php`

```php
<?php
return array(
    'max_token' => '…',
    'notify_user_id' => 12345678,  // > 0 — приоритетнее chat
    // 'notify_chat_id' => 0,       // если шлём в чат, user_id обнулить
);
```

Скрипт деплоя в Gidra генерирует только **`max_token`** и **`notify_user_id`**. Для `notify_chat_id` либо дописать генератор в `deploy-beget.sh`, либо один раз отредактировать файл на сервере (при следующем деплое перезапишется, если не расширить скрипт).

## 6. Контракт фронта → PHP

**Метод:** `POST`  
**Content-Type:** `application/json`

```json
{
  "name": "Имя",
  "phone": "+7 …",
  "comment": "",
  "company": "",
  "page": "/path?query"
}
```

- **`company`** — honeypot: для людей скрыто (`sr-only`); боты часто заполняют → ответ `200 {"ok":true}` **без** MAX.
- Ошибки валидации: `400` + `{"ok":false,"error":"…"}`.
- Нет секретов / неверная конфигурация: `503`.
- Ошибка связи с MAX: `502`.

---

## 7. Контракт PHP → MAX

- **URL:** `https://platform-api.max.ru/messages?user_id=<id>` или `?chat_id=<id>`
- **Заголовки:** `Content-Type: application/json`, `Authorization: <токен>`
- **Тело:**

```json
{
  "text": "…markdown…",
  "format": "markdown",
  "notify": true
}
```

## 8. Спам-фильтр (`detect_spam_lead`) — перенос без переделки

Функция целиком живёт в **`submit-lead.php`**: `detect_spam_lead($name, $phoneNorm, $comment)` и вспомогательная **`utf8_lower()`**.

**Возврат:** `array('spam' => bool, 'tags' => array('краткие признаки'))`.

### 8.1. Жёсткие правила (сразу `spam => true`)

| Условие | Тег в сообщении |
|---------|-----------------|
| В **имени** есть признаки ссылки: `http`, `www.`, `t.me/`, `bit.ly`, `tinyurl`, `cutt.ly` | ссылка в имени |
| **≥ 3** ссылок в имени + комментарии (по regex) | много ссылок |
| Телефон РФ 11 цифр, **все одинаковые** (`79999999999`) | телефон из одинаковых цифр |
| Подстрока из списка **strong** (ниже) в объединённом нижнем регистре текста | запрещённая тематика |

**Список strong (фрагменты, `strpos`):**  
`viagra`, `cialis`, `casino`, `pornhub`, `forex`, `binary opt`, `crypto invest`, `bitcoin invest`, `nft drop`, `казино`, `виагра`, `порно`, `эскорт`, `интим услуг`, `заработок без вложен`, `пассивный доход`, `seo продвижен`, `накрутка подпис`, `кредит всем`, `займ без отказ`, `mlm`, `финансовая пирамида`, `удалённая работа на дому`, `удаленная работа на дому`.

### 8.2. Балльная система (порог **55**)

| Сигнал | Баллы | Тег |
|--------|-------|-----|
| **≥ 1** ссылка в тексте (имя+комментарий) | +58 | ссылка в сообщении |
| Номер `79000000000`, `71111111111`, `70000000000` | +45 | типовой фейк-номер |
| Любая фраза из списка **weak** | +42 | реклама/маркетинг |
| Один символ **≥ 7** раз подряд в имени или комментарии | +22 | повторы символов |
| Имя **≥ 12** символов, только латиница/пробелы/точки/дефис, без кириллицы, и есть ≥1 ссылка | +25 | латиница + ссылка |

**Weak:** `купить подписчик`, `раскрутка инстаграм`, `продвижение в топ гугл`, `продвижение в топ google`, `telegram investment`, `получите бонус`, `увеличить продажи за 3 дня`.

Итог: `$isSpam = ($score >= 55)`; теги дедуплицируются; если спам без тегов — подставляется `эвристика`.

### 8.3. Поведение в уведомлении

Перед блоком заявки добавляются строки:

```text
⚠️ **SPAM** — <теги через запятую>

```

### 8.4. Настройка под другой проект

- **Меньше ложных SPAM** из‑за одной ссылки: уменьшить вес ссылки (сейчас **58**) или поднять порог **55**.
- **Больше перехвата:** добавить фразы в `strong` / `weak`, или снизить порог.
- **Другая страна телефонов:** заменить `normalize_phone_ru` и проверки номера на нужный формат.

При переносе скопируйте **целиком** функции `detect_spam_lead` и `utf8_lower` и вызов перед сборкой `$lines` (как в текущем `submit-lead.php`).

## 9. PHP на хостинге

Код рассчитан на **PHP 5.6+** (старый дефолт на части shared-хостингов). Рекомендуется **PHP 8.x** — меньше сюрпризов с `json_*` и curl.

### Beget (gidravlika64.ru): панель и реальная версия в браузере

На Beget **две разные картины**:

| Где | Что видно |
|-----|-----------|
| **SSH**, команда `php -v` | Часто остаётся **старый CLI** (`/usr/bin/php`, например 5.6) — это нормально для аккаунта. |
| **Веб** (`curl -I` на `*.php`, заголовок `X-Powered-By`) | То, что реально обрабатывает сайт. |

Если в [панели Сайты](https://cp.beget.com/sites) выбран **PHP 8.4**, а `X-Powered-By` всё равно **PHP/5.6.x**, добавьте в **`public_html/.htaccess`** (в корне сайта, рядом с `index.html`) строку:

```apache
AddHandler application/x-httpd-php84 .php
```

После деплоя проверка:

```bash
curl -sI "https://ВАШ_ДОМЕН/api/submit-lead.php" | grep -i powered
# Ожидается: x-powered-by: PHP/8.4.x
```

Если после добавления `.htaccess` ответ **500** — закомментируйте или удалите строку `AddHandler` и уточните в поддержке Beget привязку домена к версии PHP.

**Cron** на Beget: указывайте полный путь к нужному интерпретатору, например:

`/usr/local/php/cgi/8.4/bin/php` или актуальный путь из раздела «Информация о сервере» в панели (в FAQ встречается префикс `/usr/local/php-cgi/ВЕРСИЯ/bin/php`).

## 10. Чеклист для нового проекта

- [ ] Скопировать `public/api/submit-lead.php`, поправить домен/бренд в тексте уведомления.
- [ ] Скопировать `lead-secrets.example.php` → на сервер как шаблон; реальный `lead-secrets.php` не коммитить.
- [ ] Фронт: форма с `company` (honeypot), `POST` на тот же путь, обработка `ok` / ошибок.
- [ ] Создать бота в MAX, взять токен, выполнить `/start` с целевого аккаунта.
- [ ] `curl` к `/updates`, записать `user_id` в секреты.
- [ ] Настроить выгрузку `lead-secrets.php` (как `.deploy/.max.env` + скрипт деплоя или ручная заливка).
- [ ] Проверить: успешный `POST` → сообщение в MAX; honeypot → `ok` без сообщения; заведомый спам → префикс **SPAM**.

## 11. Типичные сбои

| Симптом | Что проверить |
|---------|----------------|
| `503` «не настроен lead-secrets» | Файл не создан или нет прав на чтение в `api/`. |
| `503` «Неверная конфигурация уведомлений MAX» | Пустой токен или оба id = 0. |
| `502` | Сеть, неверный токен, MAX недоступен, неверный `user_id`/`chat_id`. |
| Сообщений нет, `ok: true` | Сработал honeypot (`company` не пустой). |

## 12. Тот же бот MAX в портфолио **sii5** (Netlify, без PHP)

Сайт **sii5** шлёт заявки в **`/.netlify/functions/max-lead-mirror`**, функция вызывает **`POST https://platform-api.max.ru/messages`** с тем же токеном и логикой «сначала **user_id**, иначе **chat_id**», что и **`submit-lead.php`** в Gidra.

### Перенос значений из Gidra (без коммита секретов в git)

Файл **`Gidra/.deploy/.max.env`** (локально, не в git) обычно содержит:

| В `.max.env` (Gidra) | В Netlify → Environment variables (sii5) |
|----------------------|---------------------------------------------|
| `MAX_BOT_TOKEN=…` | **`MAX_BOT_TOKEN`** — то же значение |
| `MAX_NOTIFY_USER_ID=…` | **`MAX_NOTIFY_USER_ID`** или **`MAX_USER_ID`** — то же число (не **`0`**) |
| при необходимости `notify_chat_id` из **`lead-secrets.php` на сервере** | **`MAX_CHAT_ID`** или **`MAX_NOTIFY_CHAT_ID`** |

Пустое или **`0`** для user/chat в Netlify означает «не использовать» (как в PHP).

После внесения переменных — **Deploy** (желательно **Clear cache and deploy**). Подробнее — **`max_bot_instrukt.md`**, раздел про sii5.

### 12.1. Заявка в Netlify Forms есть, в MAX — нет

| Что проверить |
|---------------|
| **Netlify → Site → Logs → Functions** для `max-lead-mirror`: статус **503** — не заданы **`MAX_BOT_TOKEN`** и пара **`MAX_USER_ID` / `MAX_NOTIFY_USER_ID`** или **`MAX_CHAT_ID` / `MAX_NOTIFY_CHAT_ID`** (пусто и **`0`** считаются «не задано»). |
| **401 / 403 / 4xx** от `platform-api.max.ru` — токен отозван, неверный **`user_id`**, пользователь не нажал **Старт** у бота. |
| В браузере на **опубликованном сайте** (не только localhost): после отправки формы в консоли не должно быть **`max-lead-mirror`** с кодом **≠ 200**; при сбое MAX показывается жёлтое предупреждение под текстом «Спасибо». |
| Отправка без JavaScript (**только** Netlify Forms на `/spasibo/`) **не** вызывает функцию — уведомление в MAX только при работающем JS на странице формы. |
