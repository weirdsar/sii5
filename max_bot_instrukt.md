# Инструкция по ботам MAX (РазумныеОкна / проект vpn)

Документ в двух частях: **для пользователя/заказчика** (кабинет MAX, токен, группа) и **для разработчика/ИИ** (код, кнопки, AmoCRM, отладка).

Официальная документация MAX: [dev.max.ru — API](https://dev.max.ru/docs-api), [подготовка бота](https://dev.max.ru/docs/chatbots/bots-coding/prepare).

---

## Часть A. Инструкции для пользователя

### A1. Где всё настраивается

1. Кабинет для бизнеса: **[business.max.ru](https://business.max.ru/self)** (раздел **Чат-боты** и **Интеграция**).
2. Сайт заявок: PHP-скрипты в каталоге **`api/`** на хостинге (рядом — **`lead-secrets.php`**, в git не коммитится).
3. Группа менеджеров в MAX (например **«Заявки-окна»**): туда падают копии заявок с сайта; под сообщением — кнопки статусов.

### A2. Создание бота и получение токена

1. В кабинете MAX создайте или выберите **чат-бота** для организации.
2. Дождитесь **модерации** бота, если платформа её требует (без этого интеграция может быть недоступна).
3. Откройте **Интеграция** (или аналог в интерфейсе) и скопируйте **токен доступа API** (длинная строка для заголовка `Authorization`).
4. Токен передаётся **только** на ваш сервер (файл `lead-secrets.php`), **не** публикуйте его в чатах, скриншотах и репозитории.

### A3. Первый контакт с ботом (важно для доставки)

С аккаунта, **которому бот будет писать**, откройте бота в приложении MAX и нажмите **«Старт»** или отправьте **`/start`**. Иначе у платформы могут быть ограничения на инициацию диалога ботом (актуальные правила — в документации MAX).

### A4. Группа «Заявки-окна» и роль бота

1. Создайте **групповой чат** или используйте существующий.
2. Добавьте в группу **бота** как участника.
3. Назначьте бота **администратором** группы.  
   **Зачем:** сервер должен получать события **`message_created`** (обычные текстовые сообщения менеджеров в группе). Без прав администратора в группе эти события на webhook часто **не приходят**, и не работают сценарии **«Расчёт»** и **«Комментарий»** (ответ текстом в чат после нажатия кнопки).  
   Кнопки (**`message_callback`**) могут работать и без админки — это отдельный тип обновлений.

### A5. Узнать `chat_id` группы (один раз)

- Удобный способ: метод API **`GET https://platform-api.max.ru/chats`** с заголовком `Authorization: <токен бота>` — в списке чатов найти группу по названию и скопировать **`chat_id`** (у групп id может быть **отрицательным** — это нормально).
- Альтернатива: **`GET /updates`** и события после сообщения в группе — в теле события ищут `chat_id` в получателе.

Значение записывается в **`lead-secrets.php`** в поле **`notify_chat_id`**. Для отправки **только в группу** обычно **`notify_user_id`** ставят в **`0`**.

### A6. Что уже делает сайт (без участия пользователя в коде)

- Заявки с форм уходят в Telegram (если включено в dc-leads) и **параллельно** в MAX в указанный чат/пользователя — см. плагин **dc-leads-max-mirror** и **`api/submit-lead.php`** (статические формы).
- После отправки формы клиенту могут открываться **канал компании** и **чат с ботом** по ссылке с `?start=<токен>` — склейка **`user_id`** с заявкой через **`api/max-webhook.php`** и событие **`bot_started`**.

### A7. Webhook (обычно настраивает разработчик)

Подписка **`POST /subscriptions`**: URL вида `https://<ваш-домен>/api/max-webhook.php`, типы обновлений минимум **`bot_started`**, **`message_callback`**, **`message_created`**, секрет — строка из **`lead-secrets.php`** (`max_webhook_secret`). MAX присылает её в заголовке **`X-Max-Bot-Api-Secret`**.

### A8. Ограничения, о которых стоит знать

- Удаление сообщений в группе после сохранения комментария делается через API **`DELETE /messages`**; у платформы часто есть лимит **по времени** (например, удаление только для сообщений **младше ~24 часов**).
- Текст заявок и кнопки завязаны на **один токен бота** и **одну** группу из `notify_chat_id` (если не менять конфиг).

## Часть B. Инструкции для разработчика / ИИ-агента

### B1. Карта системы (Репозиторий Razokna + api)

| Компонент | Файл / место | Назначение |
|-----------|----------------|------------|
| Секреты | `Razokna/api/lead-secrets.php` (на сервере, не в git) | `max_token`, `notify_chat_id`, `max_webhook_secret`, опционально AmoCRM, `max_channel_url`, `max_bot_username`, флаг `max_group_buttons` |
| Заявки с WP-форм | `dc-leads` + mu-plugin **`dc-leads-max-mirror.php`** | Зеркало в MAX на хуке **`dc_action_leads_send_crm`** при включённом **`send_telegram`** |
| Кнопки и логика статусов | **`includes/razokna-max-group-buttons.php`** | Клавиатуры, callback, `message_created`, AmoCRM notes/move |
| Webhook | **`api/max-webhook.php`** | Проверка секрета, разбор updates, `message_callback`, `message_created`, **`bot_started`** |
| Склейка заявка↔бот | **`razokna-max-lead-link.php`**, **`app.js`** | Токен 32 hex, transient, окна канал+бот |
| Шаблон текста заявки | **`max.obrazec.md`** (корень vpn) | Соглашения по формату строк |

Константы AmoCRM в коде кнопок (проверять в файле при смене воронки):

- **`RAZOKNA_AMO_OKNA_PIPELINE_ID`** — воронка «Окошки».
- **`RAZOKNA_AMO_OKNA_PRIMARY_STATUS_ID`** — этап «Первичный/срочные» (перевод при **«В работе»**).
- **`RAZOKNA_AMO_STATUS_CLOSED_LOST`** — закрыто не реализовано (при отказе с причиной).

### B2. `lead-secrets.php`: поля, которые реально используются

- **`max_token`** — Platform API (и `Authorization`; при необходимости без префикса `Bearer ` — код часто обрезает).
- **`notify_user_id` / `notify_chat_id`** — куда слать заявки (группа: `notify_chat_id ≠ 0`, `notify_user_id = 0`).
- **`max_webhook_secret`** — MUST совпадать с подпиской webhook и заголовком на `max-webhook.php`.
- **`max_channel_url`**, **`max_bot_username`** — для UI после формы (диплинк бота).
- **`amocrm_base_url`** или **`amocrm_subdomain`** + **`amocrm_long_token`** — ссылки в тексте заявки, заметки из кнопок, поиск сделки по телефону/маркеру (см. код зеркала).
- **`max_group_buttons`** — `false` отключает inline-клавиатуру под заявкой (по умолчанию включено).

### B3. Поток заявки → сообщение в MAX с кнопками

1. **`dc_filter_leads_data_amo`** добавляет маркер **`[mx…]`** в имя сделки (для поиска в Unsorted).
2. Отправка в MAX **откладывается на `shutdown`**: ждёт появления сделки в AMO, подставляет ссылку **`…/leads/detail/<id>`** в текст.
3. Регистрируется transient **`razokna_mgb_<16hex>`** через **`razokna_max_gb_register_lead`**, к сообщению прикрепляется **`razokna_max_gb_keyboard_attachment($token)`** (если **`razokna_max_gb_enabled`**).

TTL transient заявки: **45 суток** (`RAZOKNA_MAX_GB_TTL`). Ожидание текста для «Расчёт»/«Комментарий»: **30 минут** (`RAZOKNA_MAX_GB_CALC_WAIT_TTL`).

### B4. Все кнопки: payload и поведение

Формат: **`<код>|<token>`** или **`<код>|<token>|...`**. Токен — **16 hex**.

| Код | Подпись кнопки | Действие |
|-----|----------------|----------|
| **`t`** | В работе | Статус «взято», строка в карточке, **перевод сделки** в воронку Окошки → этап первичный (`razokna_max_gb_amo_move_to_primary`). |
| **`c`** | Расчёт | Включает ожидание **одного** текстового сообщения в чате; затем фиксируется расчёт + «комментарий к расчёту», заметки AMO, **PUT** текста сообщения заявки. Transients: `razokna_mgb_cw_*`, `razokna_mgb_cwu_<uid>`. |
| **`k`** | Комментарий | Ожидание **одного** сообщения; строки в **`chat_notes`**, заметка AMO «Комментарий (чат)», обновление сообщения. Transients: `razokna_mgb_nw_*`, `razokna_mgb_nwu_<uid>`. |
| **`m`** | Замер | Показ клавиатуры дат **`md`**. |
| **`md`** | дата (Сегодня / …) | Показ слотов времени **`mt`**. |
| **`mt`** | интервал времени | Фиксация замера, заметка AMO, обновление текста. |
| **`rn`** | нет ответа | Лог попыток, статус, заметка AMO. |
| **`r`** | отказ | Показ клавиатуры причин **`rj`**. |
| **`rj`** | Спам / Цена / Другое | Отказ, заметка, перевод в **closed lost** при успешном PATCH. |
| **`d`** | СДЕЛКА | Фиксация сделки в тексте (без доп. этапа AMO в базовом сценарии). |

Взаимоисключение: при нажатии **`c`** сбрасывается режим **`k`** на этой заявке и наоборот (meta + transients).

Ответы на callback: **`POST https://platform-api.max.ru/answers?callback_id=...`** (`razokna_max_gb_send_answer` / `_formatted`).

### B5. Событие `message_created`

Обрабатывает **`razokna_max_gb_handle_message_created`**:

1. Игнор ботов, пустой текст, команды с `/`.
2. Сначала ветка **«Расчёт»** (если в meta есть **`calc_comment_pending`**).
3. Иначе ветка **«Комментарий»** (**`chat_note_pending`**).
4. После успешного **PUT** карточки или **успешного** fallback **POST** в чат — попытка **`DELETE /messages?message_id=`** для входящего сообщения пользователя (**`razokna_max_gb_delete_message`**).

Лог отладки расчёта: **`ABSPATH/api/max-calc-debug.log`** (строки вроде `ok_calc`, `calc_wait_start`, `note_wait_start`).

### B6. AmoCRM: что вызывается из кнопок

- **`razokna_max_gb_sync_status_line_to_amo`** — парсит **`amocrm.ru/leads/detail/<id>`** из превью текста заявки, добавляет **common note** через **`/api/v4/leads/<id>/notes`**.
- Перевод этапа: **`PATCH /api/v4/leads/<id>`** с `pipeline_id` / `status_id` (константы в начале `razokna-max-group-buttons.php`).
- Поиск/привязка лида при отправке в MAX — логика в **`dc-leads-max-mirror.php`** (unsorted, маркер, телефон).

При смене воронки/этапов в Amo — **обновить константы** или вынести в конфиг.

### B7. Формы сайта и dc-leads

- Зеркало MAX срабатывает только если в опциях dc-leads включено **`send_telegram`**.
- Поля формы проходят тот же пайплайн, что и Telegram; текст для MAX собирается по правилам зеркала и **`max.obrazec.md`**.
- Отдельно: **`submit-lead.php`** для не-WP заявок — свой путь, тот же `lead-secrets.php`.

### B8. Чеклист при поломке

1. **`lead-secrets.php`** читается, токен не пустой, `notify_chat_id` верный.
2. Webhook **403** — неверный **`X-Max-Bot-Api-Secret`** или пустой секрет.
3. Кнопки есть, текст после «Расчёт» не обрабатывается — бот **не админ** в группе или нет **`message_created`** в подписке.
4. Нет заметок в AMO — в тексте заявки нет ссылки на лид или не настроены **`amocrm_*`** токены.
5. **`error_log`** PHP и **`max-calc-debug.log`** на сервере.

### B9. Файлы, которые трогать при изменении логики кнопок

- Основной: **`Razokna/wp-content/mu-plugins/includes/razokna-max-group-buttons.php`**
- Webhook: **`Razokna/api/max-webhook.php`**
- Пользовательская документация по развёртыванию: **`MAX_LEADS_SETUP.md`**

## Сайт-портфолио sii5 (Astro, Netlify)

Здесь **нет PHP в репозитории**: заявки с **`ContactForm`** / **`BriefForm`** уходят в **Netlify Forms** и **параллельно** в MAX через Netlify Function **`/.netlify/functions/max-lead-mirror`** (серверный `fetch` — **без CORS** в браузере).

### Чеклист развёртывания (вручную в Netlify)

**Рекомендуется: заявки сразу в чат MAX** (без своего сервера). Функция вызывает **`POST https://platform-api.max.ru/messages`** ([документация](https://dev.max.ru/docs-api/methods/POST/messages)).

1. **Environment variables** (все нужные контексты: Production и при необходимости Deploy Previews):
   - **`MAX_BOT_TOKEN`** — токен из [business.max.ru](https://business.max.ru/self) → **Чат-боты** → **Интеграция** (в **`Authorization`**, как в доке MAX).
   - **`MAX_USER_ID`** или **`MAX_NOTIFY_USER_ID`** — личные уведомления (имя **`MAX_NOTIFY_USER_ID`** совпадает с **`Gidra/.deploy/.max.env`** — можно **скопировать те же значения** в Netlify).
   - **или** **`MAX_CHAT_ID`** / **`MAX_NOTIFY_CHAT_ID`** — группа (id может быть отрицательным). Если заданы **и** user, **и** chat, используется **user** (как в **`Gidra/public/api/submit-lead.php`**). Значение **`0`** в env не считается заданным.
2. Бот должен быть **участником** этой группы; по правилам MAX пользователю иногда нужно написать боту **`/start`** до того, как бот сможет писать в личку (см. раздел **A3** выше).
3. **Deploy** → при первом включении функции желательно **Clear cache and deploy site**.
4. **Проверка**: DevTools → **Network** → отправка формы → **`max-lead-mirror`** ответ **200**. **503** — не заданы **`MAX_BOT_TOKEN`** и пара **chat/user id**, и не настроен вебхук (см. ниже). Ошибки **401/403** от MAX — проверить токен и права бота в чате.

**Альтернатива — свой HTTPS-вебхук** (если нужен PHP/CRM): **`MAX_LEAD_WEBHOOK_URL`** или **`PUBLIC_MAX_BOT_URL`**; опционально **`MAX_LEAD_WEBHOOK_BODY_FORMAT=form`** для **`application/x-www-form-urlencoded`**. Если заданы **и** токен MAX, **и** вебхук, используется **прямой MAX** (приоритет).

### CLI (`netlify link` выполнен)

```bash
# Скопируйте значения из Gidra/.deploy/.max.env (не публикуйте в git):
npx netlify env:set MAX_BOT_TOKEN "<из_MAX_BOT_TOKEN>"
npx netlify env:set MAX_NOTIFY_USER_ID "<из_MAX_NOTIFY_USER_ID>"
npx netlify deploy --prod --build
```

Локально **`astro dev`**: функция **`max-lead-mirror`** не поднимается — тест прямого MAX через **`netlify dev`** или только после деплоя на Netlify.
