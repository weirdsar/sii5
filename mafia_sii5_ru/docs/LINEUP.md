# Рассадка игроков за столом на сайте

## Основа: первая игра (раунд 1)

Зафиксированный **принцип**, от которого строятся протокол и дальнейшие расширения:

1. **Матрица** **`LINEUP_SEATS[индексКоманды][индексИгры]`** — в ячейке **номер места за столом** от **1** до **10** для этой пары в этой игре. Индексы с нуля: **`индексИгры === 0`** соответствует **первой игре** (в шапке таблицы на сайте — столбец «1»).
2. **Свойства столбца первой игры**: в столбце `g = 0` — перестановка всех десяти мест (каждое место занято ровно одной командой). То же для остальных столбцов матрицы.
3. **Строки на сайте** — команды **пары 1…10** в порядке **`lineupTeams`** (сортировка по полю **`n`** в **`pairs.ts`**). Это порядок **пар**, а не порядок **мест за столом**.
4. **Протокол / PDF (`/igra.pdf` на сайте, файл в репозитории `public/igra.pdf`, копия в `docs/igra.pdf` после генерации)**:
   - **`npm run generate:igra-pdf`** собирает файл из **19 страниц**: листы **1…18** — рассадка по играм (на странице **`k`** столбец **`LINEUP_SEATS[t][k − 1]`**); лист **19** — **чистый бланк** шаблона без программного текста;
   - на каждой странице строки **колонки `Игрок`** сортируются по месту за столом (1 → 10) для этой игры;
   - исходный файл **`docs/Protokol_igry_odnostoronii_774_novyi_774.pdf`** — накладываются только текст и координаты из мастера **`src/protocolPdfMaster.ts`** (шапка и колонка «Игрок»); **геометрию и кегли мастера не менять** без отдельного решения по макету;
   - **`docs/protokol.MD`** на генерацию PDF **не влияет** (можно оставить как заметки вручную);
   - ячейка **`Игрок`** (утверждённый вид, **`PROTOCOL_PDF_MASTER_VERSION`** + константы **`PLAYER_COLUMN`**): единый кегль **`«команда»`** на все 10 строк; **сначала** подбор так, чтобы название команды было **в одну строку** у всех пар (если геометрически нельзя — до двух строк); строка **`A - B`** всегда **крупнее** команды (кегль фамилий ≥ кегль команды + **`minPlayerFontAboveTeamPt`**), внутри остатка высоты ячейки;
   - подписи пар — **`src/lineupPdfTeams.ts`** (с **`pairs.ts`** при смене состава);
   - скрипт: **`npm run generate:igra-pdf`** → **`scripts/generate-igra-pdf.ts`**, шрифт **DejaVu Sans** (`dejavu-fonts-ttf`).
5. **Игры 2…18**: те же правила, другой индекс столбца **`g = 1 … 17`**. Любые новые бланки (вторая игра, весь турнир и т.д.) **наследуют** этот принцип: та же матрица, тот же смысл ячейки, при необходимости — та же сортировка по месту для «снимка за столом».
6. **Воспроизводимость**: один **`LINEUP_RANDOM_SEED`** в **`src/lineupGridCore.ts`** — одна и та же матрица на сайте, в PDF и в любом коде, который вызывает **`buildRandomizedSeatsGrid()`**.

Полный бланк протокола (судьи, фолы и т.д.) — отдельный файл, например **`docs/Protokol_igry_*.pdf`**; блок рассадки в нём должен **согласовываться** с пунктами выше.

---

- **Прямая ссылка на раздел:** к URL страницы добавьте **`#rassadka`** (например `https://mafia.sii5.ru/#rassadka` или с вашим `base`).
- Данные на сайте: **`src/lineupSchedule.ts`** — **`LINEUP_SEATS`**; алгоритм и seed — **`src/lineupGridCore.ts`**.
- **PDF бланков:** **`npm run generate:igra-pdf`** → **`public/igra.pdf`** (+ **`docs/igra.pdf`**), **19 страниц** (18 игр + чистый лист); скачивание: **`https://mafia.sii5.ru/igra.pdf`** (при `base` `/`); макет — **`src/protocolPdfMaster.ts`** (**`PROTOCOL_PDF_GAME_COUNT`**, **`PROTOCOL_PDF_BLANK_TAIL_SHEETS`**).
- **Блок таблицы на странице** по умолчанию скрыт. Кнопка «Показать / скрыть таблицу» отправляет пароль на **`GET`/`POST`** **`/api/lineup-table-visibility`** (с учётом `base`: см. **`assetUrl`** в **`src/lineupUi.ts`**; при необходимости переопределите **`VITE_LINEUP_TABLE_VISIBILITY_API`**). Состояние **общее для всех посетителей**: после успешного показа таблица видна у всех, пока кто-то с паролем не скроет её снова. Опрос **`GET`** каждые **8** с.
  - **Cloudflare Pages:** в каталоге **`functions/api/lineup-table-visibility.js`** — функция читает/пишет KV. В проекте Pages создайте namespace KV и привяжите binding с именем **`LINEUP_TABLE_KV`**. Переменная окружения **`LINEUP_TABLE_PANEL_PASSWORD`** (секрет) — если не задана, пароль по умолчанию **`8888`** (на сервере и в клиентском fallback при ответе **503** / без API).
  - **Без KV (статический хостинг, `vite dev`):** клиент хранит флаг в **`localStorage`** (ключ в **`lineupUi.ts`**) и синхронизирует вкладки событием **`storage`** — для «всех пользователей сайта» это не подходит, только для локальной проверки.
- Скрытие/показ блока таблицы **не** подменяет правила чисел в ячейках — они по-прежнему от URL ниже.
- Показ чисел в таблице рассадки:
  - **Без секрета в сборке:** в адресе **`?lineup=1`** или **`showLineup=1`**, также **`lineup=true`** / **`yes`** / **`show`**. Скрыть: убрать параметр или **`lineup=0`** / **`hide`** / **`false`**.
  - **С секретом:** в Cloudflare / Vite задать **`VITE_LINEUP_REVEAL_KEY`** (строка). Тогда в URL только **`?lineup=<точное значение>`** (или то же в **`showLineup`**) показывает таблицу рассадки; иначе ячейки «—».
